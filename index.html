What Defines a Successful Kickstart? Can You Predict a Failure?¶

Analyzing Kickstarter data to visualize and predict the success or failure of a campaign.

Author: Lucas Ference

Introduction:¶

Recently, crowdfunding has taken off. This is evident with the massive success of a website called Kickstarter. Anyone that has an idea for a company or product can start a campaign on Kickstarter, where the public can help fund their project. Ultimately, the campaign can fail or succeed, depending on if the goal was reached or not. Supporters (also called backers) can "pledge" a predefined amount of money, which is decided by the campaign creator. For example, a project may have three payment "packages". Below is a picture of some of the packages for a project called LUNEdot Candle.

By pledging to either of these packages, the backer should expect to receive what is described in the package if the project ends up being successful.

Using detailed data about over 300,000 kickstart campaigns in 2018, we will examine successful campaigns and compare them to campaigns that have failed. Hopefully after reading, you will be more informed about Kickstarter, and what types of signs can lead to a campaign failing or being successful.

Dataset Source¶

The dataset used includes information regarding over 300,000 kickstarter campaigns. It includes the category, the deadline and time launched, its current state (failed, canceled, success, live, or suspended), how many backers it had/has (amount of people donating to the campaign), and the total amount pledged.

This dataset was found on Kaggle. The curator of the dataset also converted all amounts spent into USD, which makes our lives a little easier since we will not have to convert any of the data.

Kaggle dataset: https://www.kaggle.com/kemical/kickstarter-projects#ks-projects-201801.csv

We have now calculated the difference percentage for each project. We can now group by each category so we can compare these percentages across categories. There is one problem however, comparing across the categories directly could give innacurate results. There is a wide range in the total projects for each category, as well as a wide range in goals.

Because of this, we will standardize the percentage by calculating the Z score for each row by first getting the average percentage and standard deviation. After calculating the Z score for every row we will average the Z scores for each category. For more information regarding Z scores and standardization, go here

In [107]:
# Create a table for calculating the Z scores.
calc = difference.copy()

# Calculate the average percentage.

# Calculate the standard deviation of diff_pct.
std_dev = calc['diff_pct'].std()

calc['z_score'] = 0

# Calculate the Z score for each project.
calc['z_score'] = (calc['diff_pct'] - avg_diff) / std_dev

z_scores = calc.groupby('category').mean().reset_index()[['category', 'z_score']]

Pledge Data vs. Success¶

Logically it makes sense that the more pledges a project has, the more likely that project is to succeed. I would like to confirm this belief by analyzing the success percentage vs. the average amount pledged. Instead of focusing on the categories and how they may differ, I will be focusing on "buckets" of data in the table, and how that "bucket" is correlated to success.

Below I have listed the steps I will be taking to perform this.

1) Order the kickstarts by average pledge amount per backer.
2) Put every ~one thousand rows into a "bucket" and calculate the win percentage of that bucket.
3) For every ~one thousand sum the total pledge amound and divide by the total number of backers to get the average pledge amount for the bucket.
4) Plot that vs. the success percentage to find a relation.

Step 1: order the kickstart data

Step 3: Get average donation amount per bucket.

In [115]:
# Break the table into 300 different sections.
binned = np.array_split(ordered, 300)
cut = pd.DataFrame(columns=['success_pct', 'bucket_avg'])

# For each section, calculate the success percentage and add to ordered table.
for df in binned:
    pct = 100 * df['success'].mean()
    pledged = df['pledged'].sum()
    backers = df['backers'].sum()
    avg = 0.0
    if backers != 0:
        avg =  pledged / backers
    cut = cut.append({'success_pct': pct, 'bucket_avg': avg}, ignore_index=True)

What is also very peculiar is that the success percentage actually starts to decrease after a ~200 bucket average! It is not a dramatic decrease, but it is something to notice. Could it be that higher goals are associated with higher average spent per backer? We will repeat the same process as above, but sorted by goal rather than amount per backer.

Null Hypothesis¶

Before we start, we should define our null hypothesis.

None of the attributes attributes in our selected table have any effect on the final state of the project.

Because we are attempting to predict a 0 or a 1 (success or failure), it would be best to use logistic regression. Logistic regression can be used to predict dependent binary variables. To read up further on this type of regression, click here.

Conclusion¶

From our exploratory analysis, we were able to see that the goal and average amount per backer were significant in determining the success of a project. We found a negative correlation between goal and success, which was significant, especially for our logistic regression model.

From our category analysis, we were able to determine that various categories possess different properties and have different success rates. Some categories had a signifciantly higher success rate than others.

The most interesting data observed, in my opinion, was from sorting based on average spent per backer, and then dividing up into buckets. By doing this, we were able to see the correlation directly between having a high average amount per buyer and success rate. This also helped us uncover a relation between goal and success, which turned out to be more significant than we thought.

With some of this additional insight into Kickstarter data, a potential user could choose his/her project based on some of the data provided above. If you are looking to start a successful project, there are certain "rules" that if followed, can help increase the chances of success. If you set a reasonable goal and choose the correct category, your chance of being successful will likely increase. Obviously, a good project will go much farther than a mediocre project, but a more educated choice can be made if the data is followed correctly.
